업무정의서 (Requirements Definition Report)

한국해양대학교 승선생활관 통합 정보 시스템 구축

1. 문서 개요
1.1 작성 목적

본 문서는 한국해양대학교 승선생활관의 생활 정보, 공지사항, 재고관리, 신청 서비스 등을 통합 제공하기 위한 승선생활관 통합 정보 시스템의 요구사항을 정의한다.
본 요구사항 문서는 개발 범위와 기능을 명확히 하여 향후 설계·개발·테스트 단계에서 기준 문서로 활용하기 위함이다.

1.2 문서 범위

본 문서는 시스템의 기능적 요구사항 및 비기능 요구사항, 사용자 정의, 화면 구성 요소를 포함한다.
UI 설계, 데이터베이스 설계 등은 본 문서의 이후 단계에서 작성 예정이다.

2. 시스템 개요
2.1 시스템 목적

한국해양대학교 승선생활관 생활관생과 사관부·생활교육지도관 등 관련 부서가 신속하고 편리하게 생활 정보를 확인하고 관리할 수 있는 통합 포털 플랫폼을 구축하는 것을 목표로 한다.

2.2 시스템 필요성

기존 생활관 정보는 여러 사이트와 공지 채널로 분산되어 있어 접근성과 편의성이 떨어진다.
본 시스템은 다음을 해결한다.

생활·학습·행정 정보를 단일 플랫폼에서 제공

공지 전달의 정확도 향상

학생 신청 및 재고 관리 업무 효율화

모바일 기반의 실시간 정보 접근성 강화

5. 비기능 요구사항
5.1 사용성(Usability)

모바일 중심 UI/UX 제공

직관적인 네비게이션 구조

응답 속도 및 화면 전환 최소화

5.2 성능(Performance)

기상/버스정보 API 실시간 연동

데이터 조회 시 2초 이내 로딩 목표

5.3 보안(Security)

HTTPS 기반 암호화 통신

관리자 기능은 로그인/인증 기반 권한 관리 적용

개인정보 보호 관련 법규 준수

5.4 유지보수성(Maintainability)

각 기능 모듈화

데이터 구조 및 API 변경에 대응 가능한 구조

5.5 확장성(Scalability)

공지사항/신청 기능 확장 가능
